# JSP 화면 접근시 주의사항
    JSP 호출할 때, GET/POST 호출을 구분하지않음.

    1) input.jsp는 GET 방식으로 호출함 -웹브라우저(주소창)에서
        ex) http://localhost:8080/calc/input.jsp?num1=&num2=

    2) input.jsp는 폼 안에서 method="post" 형식으로 해서
        SEND 버튼을 클릭시 post 호출해서 접근
        ex) <form action="calcResult.jsp" method="post">
            http://localhost:8080/Calc/calcResult.jsp

    3) 문제점은 calcResult.jsp 화면에 직접접근 = 웹브라우저로 접근(= GET 방식으로 접근)
        -> 오류가 발생함. 이유는 calcResult.jsp화면으로 넘어갈때, input화면에서
            넘어온 데이터를 읽으려고 시도함... 이 부분에서 오류가발생.

    해결책 : 다른 사람들에게, 특정의 주소, 화면 에 접근시 GET인지 POST인지 구분하고있어야함.
        --> BUT. 이런과정이 번거로움!

    차선책)
    1) JSP에서 쿼르 스트링이나 파라미터를 처리하지않고, 서블릿을 통해서 처리
    2) JSP 화면은 입력화면으로 구성하고, 처리결과를 보여주는 용도로만 사용(화면제공)
    3) 브라우저는 직접 JSP경로를 호출 하지 않고, 서블릿 경로를 통해서 JSP를 호출하는 방식을 사용.

    결론) 지금은 테스트라서 화면에 직접 접근.
        차후 서블릿이라는 자바클래스를 통해서 화면에 접근 할 예정.

    ==> JSP결과화면은 출력용으로 사용하고,
        로직 처리는 서블릿 자바 클래스로 이용하자.
